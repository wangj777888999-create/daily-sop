# 智能工作台 SOP 自动化系统 - 设计文档

**日期**: 2026-04-24
**状态**: 已批准

---

## 1. 项目概述

### 1.1 目标
将日常业务流程（数据分析、文档撰写）线上化、标准化，减少人工重复操作，提高工作效率。

### 1.2 用户
- **所有用户**: 都可以创建 SOP 和使用 SOP（无权限区分）

### 1.3 核心价值
- SOP 模板化：将业务经验固化为可复用的流程
- 自然语言交互：简单描述需求即可执行
- 本地部署：数据不上云，保护隐私

---

## 2. 系统架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户层                               │
│           所有用户（创建 + 使用 SOP）                   │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│              Web 界面 (Vue 3 - 现有项目)                │
│  SOP 管理  │  文件上传下载  │  执行结果展示            │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│              后端服务 (Python FastAPI)                  │
│  SOP 解析器  │  代码生成器  │  沙箱执行器              │
└─────────────────────────────────────────────────────────┘
                           ↓
┌───────────────────────┬─────────────────────────────────┐
│      文件系统          │        JSON 文件存储            │
│  Excel/CSV 输入输出    │   SOP 定义  │  执行记录       │
└───────────────────────┴─────────────────────────────────┘
```

### 2.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Vite + Tailwind | 复用现有项目 |
| 后端 | Python FastAPI | 轻量、易部署 |
| 数据存储 | JSON 文件 | Phase 1 使用，后续可迁移至 Notion |
| 代码执行 | subprocess 沙箱 | 隔离执行用户代码 |

**注**：未来知识库可能使用 Notion，存储层设计需考虑可迁移性。

### 2.3 分阶段建设

| 阶段 | 内容 | 周期 |
|------|------|------|
| Phase 1 | SOP 创建 + 基础执行 | 2-3 天 |
| Phase 2 | 执行历史 + 文件管理 | 1-2 天 |
| Phase 3 | 用户管理 + 高级功能 | 后续按需 |

## 3. 功能设计

### 3.1 SOP 创建方式

支持两种创建方式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **代码导入** | 粘贴 Python 代码 → 自动解析为 SOP 步骤 | 有现成代码 |
| **自然语言** | 描述步骤 → 生成可执行代码 | 从零创建 |

### 3.2 代码导入流程

1. 用户粘贴 Python 代码
2. 系统解析代码结构（函数定义、pandas 操作、文件路径等）
3. 自动提取操作步骤并生成 SOP
4. 用户可编辑/调整生成的步骤
5. 保存 SOP

**支持解析的操作类型：**
- `pd.read_excel()` / `pd.read_csv()` → 读取数据
- `df.query()` / `df[df...]` → 筛选
- `df.drop()` / `del df[col]` → 删除列
- `pd.merge()` / `df.join()` → 数据合并
- `df.groupby()` → 分组聚合
- `df.sort_values()` → 排序
- `df.to_excel()` / `df.to_csv()` → 导出

### 3.3 SOP 管理

**SOP 定义格式**:
用户一次性描述所有步骤，示例：
```
第1步：读取Excel，从"教练签到"sheet读取数据，跳过前2行
第2步：设置列名，从第3行开始为数据
第3步：筛选"课程类型"等于"学校课程"的行
第4步：删除"场馆名称"列
第5步：读取Excel，从"财务统计"sheet读取数据，跳过前3行
第6步：设置列名
第7步：转换"课程明细_已确认收入（选定时间）"为数值类型
第8步：按"课程名称"分组，求和"课程明细_已上课次（选定时间）"
第9步：合并第3步和第8步的数据，以"课程名称"为key，左连接
第10步：按"部门"排序
第11步：导出为Excel，应用格式：高亮"状态"不等于"在岗"的单元格
```

**SOP 存储结构**:
```json
{
  "id": "uuid",
  "name": "教练签到分析",
  "description": "分析教练签到数据",
  "steps": [
    {"step": 1, "action": "read_excel", "params": {...}},
    {"step": 2, "action": "filter", "params": {...}}
  ],
  "created_at": "2026-04-24",
  "updated_at": "2026-04-24"
}
```

### 3.2 SOP 执行流程

1. 用户选择 SOP
2. 用户上传数据文件
3. 系统解析 SOP，生成 Python 代码
4. 沙箱执行代码
5. 返回结果文件
6. 用户下载结果

### 3.3 代码生成与执行

**代码生成**:
将 SOP 步骤转为可执行 Python 代码：
```python
import pandas as pd
# 步骤1：读取Excel
coach = pd.read_excel('input.xlsx', sheet_name='教练签到')
# ...
```

**沙箱执行**:
- 使用 subprocess 运行
- 超时限制（默认 60 秒）
- 禁止危险操作

### 3.4 文件管理

- 上传：支持 Excel (.xlsx, .xls)、CSV
- 下载：执行结果自动打包
- 临时文件：执行后自动清理

## 4. 数据存储

### 4.1 存储策略

**Phase 1**: JSON 文件存储
- `data/sops.json` - SOP 定义列表
- `data/execution_logs.json` - 执行记录列表
- 轻量简单，无需额外依赖

**未来迁移**: 可切换至 Notion 或其他数据库

### 4.2 SOP 存储格式

```json
{
  "sops": [
    {
      "id": "uuid",
      "name": "教练签到分析",
      "description": "分析教练签到数据",
      "steps": [
        {"step": 1, "action": "read_excel", "params": {...}}
      ],
      "created_at": "2026-04-24",
      "updated_at": "2026-04-24"
    }
  ]
}
```

### 4.3 执行记录格式

```json
{
  "logs": [
    {
      "id": "uuid",
      "sop_id": "sop-uuid",
      "status": "success",
      "input_files": ["file1.xlsx"],
      "output_file": "result.xlsx",
      "created_at": "2026-04-24",
      "completed_at": "2026-04-24"
    }
  ]
}
```

## 5. API 设计

### 5.1 SOP 管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/sops | 获取所有 SOP |
| GET | /api/sops/{id} | 获取单个 SOP |
| POST | /api/sops | 创建 SOP |
| PUT | /api/sops/{id} | 更新 SOP |
| DELETE | /api/sops/{id} | 删除 SOP |

### 5.2 执行

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/execute/{sop_id} | 执行 SOP |
| GET | /api/execute/{exec_id}/status | 获取执行状态 |
| GET | /api/execute/{exec_id}/download | 下载结果 |

### 5.3 文件

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/upload | 上传数据文件 |
| GET | /api/files/{id} | 获取文件 |

## 6. 安全考虑

### 6.1 代码执行安全
- 沙箱隔离执行环境

- 禁止文件系统和网络访问

### 6.2 数据安全

- 临时文件执行后自动清理
