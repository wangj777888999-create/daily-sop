# 语义化 SOP 代码导入设计

> 版本：v1.0 · 更新于 2026-04-24

## 1. 背景与目标

### 1.1 问题

当前代码导入功能解析出的 SOP 步骤是「字面内容」而非「业务语义」：

```python
# 解析结果（当前）
Step 5: read_excel - file: 'C:\\Users\\...\\教练签到 (53).xls'

# 用户期望
Step 1: 读取教练签到数据
```

用户导入的代码通常是完成度 100% 的业务代码，路径硬编码，无法通用。

### 1.2 目标

- 解析代码时识别数据源，生成业务语义描述
- 允许配置者基于原始代码进行命名和类型标注
- 执行时支持参数化（输入上传、输出选择路径）

---

## 2. 整体架构

### 2.1 流程变化

**当前流程**：
```
代码 → AST解析 → {name, steps[{action, params}]} → 创建SOP
```

**新流程**：
```
代码 → AST解析 → 数据源识别 → 语义标注界面 → 用户确认 → 创建SOP → 执行时填参
```

### 2.2 新增模块

| 模块 | 职责 |
|------|------|
| `DataSourceDetector` | 从 AST 识别所有数据读写操作 |
| `SemanticInferencer` | 自动推断数据源类型和候选名称 |
| `SemanticAnnotationUI` | 用户标注界面（新增页面） |
| `ParametricExecutor` | 执行时参数替换 |

## 3. 数据结构

### 3.1 数据源定义

```typescript
interface DataSource {
  id: string
  name: string                    // 用户命名的业务名称
  type: 'primary' | 'reference' | 'output'
  description?: string            // 业务描述（可选）
  variableName: string            // 代码中的变量名
  operation: 'read' | 'write'
  codeSnippet: string            // 原始代码片段
  lineNumber: number
}
```

**数据类型说明**：
| 类型 | 说明 | 示例 |
|------|------|------|
| `primary` | 主数据（业务记录） | 签到记录、订单记录、财务流水 |
| `reference` | 参照表（维度数据） | 部门划分、商品分类、地区编码 |
| `output` | 输出结果 | 导出文件 |

### 3.2 SOP 步骤扩展

```typescript
interface SOPStep {
  step: number
  action: string
  params: {
    [key: string]: any
    inputSources?: string[]       // 引用的数据源 ID 列表
    outputSource?: string        // 输出数据源 ID
  }
  description: string            // 业务描述（自动生成 + 用户可编辑）
}
```

### 3.3 完整 SOP 模型

```typescript
interface SOP {
  id: string
  name: string
  description: string
  steps: SOPStep[]
  dataSources: DataSource[]       // 数据源列表
  tags: string[]
  createdAt: string
  updatedAt: string
}
```

## 4. 自动推断规则

### 4.1 数据源识别

从 AST 遍历识别以下操作：
- `pd.read_excel()` → 数据读取
- `pd.read_csv()` → 数据读取
- `df.to_excel()` → 数据输出
- `df.to_csv()` → 数据输出

### 4.2 类型自动推断

| 操作 | 默认类型 | 说明 |
|------|----------|------|
| 第一个 `read` | `primary` | 主数据 |
| 其余 `read` | `reference` | 参照表 |
| 第一个 `write` | `output` | 输出 |

### 4.3 名称自动推断

从文件路径提取：
```
"教练签到 (53).xls" → "教练签到数据"
"财务统计 (78).xls" → "财务统计数据"
"部门划分.xlsx" → "部门划分表"
```

规则：
1. 去除文件扩展名
2. 去除括号内的数字后缀
3. 添加业务后缀（数据/表/结果）

## 6. 执行时参数填参

### 6.1 执行流程

```
用户点击「执行 SOP」
    ↓
检查 SOP 的 dataSources
    ↓
发现需要用户提供的输入 → 显示「数据源配置」弹窗
    ↓
用户上传/选择文件
    ↓
系统建立：数据源ID → 实际文件 的映射
    ↓
执行 SOP（代码生成时替换为映射的文件）
    ↓
执行完成，如果需要下载 → 显示下载选项
```

```python
coach = pd.read_excel('/tmp/uploads/教练签到_uploaded.xlsx', sheet_name='教练签到')
```

## 7. 步骤描述自动生成规则

### 7.1 规则表

| action | 描述模板 | 示例 |
|--------|----------|------|
| `read_excel` | 读取【{数据源名称}】 | 读取【教练签到表】 |
| `read_csv` | 读取【{数据源名称}】 | 读取【销售记录表】 |
| `filter` | 筛选【{条件摘要}】数据 | 筛选【学校课程】数据 |
| `drop_columns` | 删除【{列名}】列 | 删除【场馆名称】列 |
| `merge` | 合并【{left_df}】与【{right_df}】 | 合并【教练数据】与【财务数据】 |
| `groupby` | 按【{by}】分组统计 | 按【课程名称】分组统计 |
| `sort` | 按【{by}】排序 | 按【部门】排序 |
| `to_excel` | 导出【{数据源名称}】 | 导出【教练签到分析结果】 |
| `to_csv` | 导出【{数据源名称}】 | 导出【汇总结果】 |

### 7.2 用户编辑

所有自动生成的描述都允许用户修改，修改后保存到 `SOPStep.description`。

## 9. 附录

### 9.1 数据源映射结构

```typescript
interface DataSourceMapping {
  [dataSourceId: string]: {
    type: 'upload' | 'path'
    file?: File           // 上传的文件对象
    path?: string        // 或指定的文件路径
  }
}

interface ExecutionConfig {
  sopId: string
  mappings: DataSourceMapping
  outputPath?: string   // 输出文件保存路径
}
```

### 9.2 术语表

| 术语 | 说明 |
|------|------|
| 数据源 | SOP 中引用的输入/输出文件 |
| 主数据 | 业务主体数据（如签到记录、订单） |
| 参照表 | 维度数据（如部门划分、商品分类） |
| 输出 | SOP 执行后生成的结果文件 |
