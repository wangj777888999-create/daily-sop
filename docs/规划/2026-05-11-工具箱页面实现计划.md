# 工具箱页面实现计划

## 一、背景与目标

### 问题
当前项目中「SOP 数据分析」页面（`/sop`）功能与实际需求不匹配，需要整体替换。

### 目标
将 `/sop` 页面改造为「**工具箱**」页面，用于展示和使用一个个独立的工具。第一个工具是已有的「课时费计算器」HTML 应用。后续可持续添加新工具。

### 核心设计决策
| 决策点 | 选择 | 理由 |
|--------|------|------|
| 工具打开方式 | 单页跳转 | 点击卡片跳转到独立工具详情页，侧边栏不变，带返回按钮 |
| 布局风格 | 响应式网格卡片 | 简洁直观，后续加工具自动扩展 |
| 工具集成方式 | iframe 嵌入 | 无需重写现有 HTML，最快落地 |
| 工具文件位置 | `public/tools/` 目录 | Vite 不处理，直接通过 URL 访问 |

## 三、接口设计 — `src/types/toolbox.ts`

```typescript
/** 单个工具的注册信息 */
export interface Tool {
  /** 唯一标识 */
  id: string
  /** 工具名称，显示在卡片上 */
  name: string
  /** 一句话描述 */
  description: string
  /** 卡片图标（emoji 或字符） */
  icon: string
  /** 主题色（hex） */
  color: string
  /** 分类标签 */
  tags: string[]
  /** iframe 加载的 URL 路径（相对于 public/） */
  src: string
  /** 展开后面板的默认高度（px），默认 800 */
  iframeHeight?: number
  /** 是否启用，默认 true */
  enabled?: boolean
}
```

const tools: Tool[] = [

### 4.3 交互细节

1. **卡片点击**：
   - 使用 `router.push('/toolbox/${tool.id}')` 跳转到工具详情页
   - 侧边栏保持不变，只有右侧内容区切换

2. **工具详情页 (ToolDetailPage.vue)**：
   - 顶部：返回按钮「← 返回工具箱」+ 工具图标和名称 + 「↗ 新窗口打开」按钮
   - 主体：iframe 加载工具 HTML，高度由 `tool.iframeHeight` 控制
   - 如果工具 ID 不存在，显示「工具不存在」+ 返回链接

3. **面包屑**：
   - Shell.vue 中动态匹配 `/toolbox/*` 路径，显示 `['工作台', '工具箱', '工具名称']`

4. **卡片样式**：
   - 复用现有 `Card` 组件（`bg-gradient-to-b from-card-bg to-card-gradient border rounded-xl shadow-card`）
   - 图标区域用工具的 `color` 做背景色 + 文字色
   - 标签用 `Chip` 组件
   - hover 效果：`shadow-card-hover` + 微上移

5. **响应式**：
   - 桌面端 3 列网格：`grid-cols-3 gap-3`
   - 平板 2 列：`md:grid-cols-2`
   - 移动端 1 列：`grid-cols-1`

### 4.4 空状态

当 `tools` 数组为空时（或所有工具 `enabled: false`），显示：
- 一个图标 + 文字「暂无可用工具」
- 提示「后续添加工具只需在工具注册表中新增条目」

## 七、工具间联动机制（框架）

在 `ToolboxPage.vue` 中搭建基础通信框架：

### 7.1 消息总线

```typescript
// 监听来自 iframe 工具的消息
window.addEventListener('message', (event) => {
  // 验证来源
  const { toolId, action, payload } = event.data
  // 路由给目标工具或执行全局动作
})
```

### 7.2 工具端发送

工具 HTML 中通过以下方式与父页面通信：
```javascript
// 发送数据给其他工具（指定 origin 而非 '*'，防止消息被其他源截获）
window.parent.postMessage({
  toolId: 'fee-calculator',
  action: 'share-data',
  payload: { /* 数据 */ }
}, window.location.origin)
```

### 7.3 共享数据层

使用 localStorage，命名空间 `toolbox_shared_`：
```typescript
// 写入
localStorage.setItem('toolbox_shared_fee_result', JSON.stringify(data))
// 读取
const data = JSON.parse(localStorage.getItem('toolbox_shared_fee_result') || '{}')
```

当前阶段只搭好框架，具体联动逻辑按需实现。
