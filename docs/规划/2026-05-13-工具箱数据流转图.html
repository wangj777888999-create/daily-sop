consolidation_output/签到数据整合

🔑 关键设计要点

1. 整合优先读取
校内月度分析：confirmed JSON → SQLite 回退

2. 校外完全隔离
不共享 SQLite，每次全量上传

3. 课程类型共享
course_types.json 被两个工具消费

4. 课时费消费校内月度
通过 Artifact API 导入，是校内管线下游
待确认 / 潜在问题

⚠ 结构性问题

1. 校外累积分析后端未实现
前端路由/tool卡片已注册，backend/tools/ 中无对应模块。

2. check_date 取当天日期而非 Excel 解析
同一天上传不同日期的数据会共用一个 check_date，按月查询可能不准。

3. 校内/校外共用 monthly_analyses 表
通过 analysis_type 字段区分，未来类型增多时需注意。

💡 优化建议

1. 签到整合可展示 unmapped_courses 提醒
每日签到处理完返回未映射课程列表，整合工具可以直接展示。

2. 校外累积消费格式需明确
校外月度输出"综合统计"Sheet，累积分析应直接消费此格式。

3. 课时费计算结果是否入库？
当前仅在前端计算展示，如后续需被其他工具引用（如薪酬系统），应考虑入库。
