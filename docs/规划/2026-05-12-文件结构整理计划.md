#### ~~B3. 补全路由定义~~ — **不适用（SOP 已废弃）**

#### B4. 后端整理（低风险）

| 操作 | 说明 |
|---|---|
| `backend/fcntl.py` → `backend/compat/fcntl.py` | 创建 `compat/` 子包，将 Windows shim 归入；更新 `storage.py` 和 `artifacts.py` 的 `import fcntl` 为 `from backend.compat import fcntl` 或调整 sys.path |
| `backend/artifacts.py` → `backend/sops/artifacts.py` | 归入 sops 子包；更新 `routes.py` 中的 import |

## 方案 C — 全面重构 src/ 层级（待执行，方案 B 之后）

> **前置条件**：方案 B 完成且测试通过。

### 目标
将当前 `src/ui/pages/` + `src/ui/components/` 的"类型分组"结构，改为"业务域分组"（feature-based）结构。

### 目标结构

```
src/
  features/
    sop/
      pages/          ← SOPList, SOPCreate, SOPImport, SOPExecute
      components/     ← ExecutionConfig（目前在 src/ui/components/sop/）
      types.ts        ← 目前的 src/types/sop.ts
    knowledge/
      pages/          ← KnowledgePage
      components/     ← DocumentCard, DocumentPreview, SearchResultCard, UploadDialog
      types.ts        ← 目前的 src/types/knowledge.ts
    toolbox/
      pages/          ← ToolboxPage, ToolDetailPage, MonthlyAnalysis（合并后）
      types.ts        ← 目前的 src/types/toolbox.ts
    home/
      pages/          ← HomePage
    analytics/
      pages/          ← AnalyticsPage（占位）
    database/
      pages/          ← DatabasePage（占位）
    policy/
      pages/          ← PolicyPage
  shared/
    components/       ← Button, Card, Chip, Input, DataTable, MetricCard, ...（目前 src/ui/components/common/）
    composables/      ← useDesignTokens（目前 src/composables/）
    types/            ← src/types/index.ts
    data/             ← src/data/tools.ts
  layout/
    Shell.vue
    Sidebar.vue
    TopBar.vue
    AIPanel.vue
  assets/
    styles/main.css
  router/
    index.ts
  stores/
    aiChat.ts, app.ts, knowledge.ts, navigation.ts
  services/
    sopApi.ts, knowledgeApi.ts
```

### 影响范围
- **约 50+ 文件**的 import 路径需要更新
- Vite 路径别名 `@` 不变，所有 import 写法仍为 `@/features/sop/...`
- `tsconfig.json` 无需改动
- 建议用 IDE 全局替换 + TypeScript 类型检查兜底

### 执行策略
1. 一次性移动所有文件（`git mv`）
2. 用 `npm run type-check` 暴露所有断掉的 import
3. 逐一修复，直到 `type-check` 通过
4. 提交
