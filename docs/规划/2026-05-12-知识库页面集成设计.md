### 4.1 改造目标

```
点击"知识库"按钮
      ↓
弹出文档选择器（modal/dropdown）
      ↓
勾选文档 → 确认
      ↓
已选文档显示在"数据源"卡片

发送对话时，携带 doc_ids 到后端（预留接口，当前后端 /generate 支持 doc_ids）
```

**方案 B：全局 DocumentPicker 组件**

复用 `src/ui/components/knowledge/` 下已有的组件，封装一个 `DocumentPicker.vue`，接收 `v-model` 返回选中 doc_ids。PolicyPage 的参考资料卡片未来也可复用。

> **建议先做方案 A**（内联实现，改动最小），待多处复用需求确认后再抽 B 方案。

### 4.4 核心逻辑（伪代码）

```typescript
// AnalyticsPage.vue script
import { useKnowledgeStore } from '@/stores/knowledge'

const kbStore = useKnowledgeStore()
const showKBPicker = ref(false)
const selectedKBDocIds = ref<string[]>([])

onMounted(async () => {
  await kbStore.loadDocuments()
})

function toggleDoc(docId: string) {
  const idx = selectedKBDocIds.value.indexOf(docId)
  if (idx >= 0) selectedKBDocIds.value.splice(idx, 1)
  else selectedKBDocIds.value.push(docId)
}

// 发送对话时，将 selectedKBDocIds 作为上下文（目前后端 chat 接口尚未实现，先保存状态）
```
