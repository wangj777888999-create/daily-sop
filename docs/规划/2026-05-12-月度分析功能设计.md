# 月度分析功能设计

> 日期：2026-05-12
> 状态：待审批

## 1. 背景与目标

用户每月需要对教培业务数据进行汇总分析，当前流程是手动在 Excel 中查看「校内分析表」（含 5 个 sheet），缺乏可视化和快速洞察。

**目标**：在智能工作台中提供月度分析工具，用户上传已组织好的 Excel 表格，系统自动解析并生成：
1. 可视化仪表盘（图表 + 指标卡片）
2. 文字摘要报告（可复制）
3. 格式化 Excel 导出

**非目标**（本次不做）：
- 月度趋势对比（跨月数据）
- 从外部数据源自动拉取数据
- 与课时费计算器的数据联动

## 2. 方案选型

采用 **方案 C：Vue 前端 + 浏览器端解析**。

| 维度 | 说明 |
|------|------|
| 前端 | Vue 组件，使用项目 design tokens，统一布局风格 |
| 解析 | SheetJS（xlsx）在浏览器端解析，不依赖后端 |
| 图表 | ECharts |
| 导出 | SheetJS 生成新 Excel |
| 数据流 | 上传 → 解析 → 聚合 → 可视化 → 导出/产出 artifact |

**理由**：
- 课时费计算器已验证 SheetJS 浏览器端解析可行
- 数据量小（~500 行），浏览器端计算无压力
- Vue 组件天然支持 artifact 输出，供其他工具消费
- 无需后端改动，纯前端实现

```typescript
// === 原始解析数据 ===

interface SchoolAnalysisRow {
  dept: string        // 部门
  school: string      // 学校名称
  course: string      // 名加课
  coach: string       // 教练姓名
  students: number    // 上课人次
  lessons: number     // 上课次数
  revenue: number     // 已确认收入
}

interface CoachAttendanceRow {
  dept: string
  school: string
  coach: string
  onDuty: number       // 在岗次数
  earlyLeave: number   // 早退次数
  absent: number       // 旷工次数
  late: number         // 迟到次数
  total: number        // 总计次数
  personalRate: number // 教练个人签到率 (0-100)
  schoolRate: number   // 学校签到率 (0-100) 仅首行有值
  deptRate: number     // 部门签到率 (0-100) 仅首行有值
}

interface TypeAnalysisRow {
  dept: string
  type: string         // 体育课/兴趣班/校队/梯队/社团课...
  typeCount: number    // 该类型课次数
  totalLessons: number // 部门总课次
  paidLessons: number  // 收费类课次
  paidRatio: number    // 收费类占比 (0-100)
  outputText: string   // 自动生成的分析文字
}

interface RefundRow {
  dept: string
  school: string
  studentCount: number // 退款学员数
  refundAmount: number // 退款金额
}

interface CheckinDetailRow {
  dept: string
  school: string
  courseType: string   // 课程类型
  courseName: string   // 课程名称
  coach: string
  date: string         // 课程日期
  startTime: string    // 开课时间
  endTime: string      // 下课时间
  signIn: string       // 签到时间
  signOut: string      // 签退时间
  status: '在岗' | '迟到' | '早退' | '旷工'
  note: string         // 备注
}

// === 聚合分析数据 ===

interface AnalysisSummary {
  totalRevenue: number       // 总收入
  totalLessons: number       // 总课次
  totalStudents: number      // 总上课人次
  totalCoaches: number       // 教练人数
  totalSchools: number       // 学校数量
  totalDepts: number         // 部门数量
  avgAttendanceRate: number  // 平均签到率
  totalRefund: number        // 总退款金额
  refundStudentCount: number // 退款学员数
  paidLessonRatio: number    // 收费课次占比
  deptSummaries: DeptSummary[]    // 按部门汇总
  coachRankings: CoachRanking[]   // 教练排名（按收入/课次/签到率）
}

interface DeptSummary {
  dept: string
  revenue: number
  lessons: number
  coaches: number
  schools: number
  attendanceRate: number
}

interface CoachRanking {
  coach: string
  dept: string
  revenue: number
  lessons: number
  students: number
  attendanceRate: number
}

// === 完整分析数据包 ===

interface MonthlyAnalysisData {
  fileName: string
  uploadTime: Date
  schoolAnalysis: SchoolAnalysisRow[]
  coachAttendance: CoachAttendanceRow[]
  typeAnalysis: TypeAnalysisRow[]
  refunds: RefundRow[]
  checkinDetails: CheckinDetailRow[]
  summary: AnalysisSummary
}
```

## 7. 解析层设计

### 7.1 合并单元格处理

SheetJS 读取合并单元格时，非首格返回 `undefined`。采用 **向前填充（forward fill）** 策略：

```typescript
function forwardFill(rows: any[][], fillColumns: number[]): any[][] {
  const result = [...rows]
  for (let i = 1; i < result.length; i++) {
    for (const col of fillColumns) {
      if (result[i][col] === undefined || result[i][col] === null) {
        result[i][col] = result[i - 1][col]
      }
    }
  }
  return result
}
```

需要 forward fill 的列：
- 校内分析：列 0（部门）、列 1（学校名称）
- 教练考勤：列 0（部门）、列 1（学校）
- 退款分析：列 0（部门）

### 7.2 数值处理

- 百分比列：去掉 `%` 后转 number（如 `"71.01%"` → `71.01`）
- 金额列：直接转 number，处理可能的空值
- 日期列：保持字符串格式，不做 Date 对象转换

### 7.3 聚合计算

```typescript
function computeSummary(data: MonthlyAnalysisData): AnalysisSummary {
  // 按部门汇总
  const deptMap = new Map<string, DeptSummary>()
  for (const row of data.schoolAnalysis) {
    const dept = deptMap.get(row.dept) || initDept(row.dept)
    dept.revenue += row.revenue
    dept.lessons += row.lessons
    // ...
  }

  // 教练排名
  const coachMap = new Map<string, CoachRanking>()
  for (const row of data.schoolAnalysis) {
    const coach = coachMap.get(row.coach) || initCoach(row)
    coach.revenue += row.revenue
    coach.lessons += row.lessons
    coach.students += row.students
  }
  const coachRankings = [...coachMap.values()]
    .sort((a, b) => b.revenue - a.revenue)

  // 签到率合并（从教练考勤 sheet 取）
  for (const ranking of coachRankings) {
    const att = data.coachAttendance.find(c => c.coach === ranking.coach)
    if (att) ranking.attendanceRate = att.personalRate
  }

  return {
    totalRevenue, totalLessons, totalStudents,
    totalCoaches, totalSchools, totalDepts,
    avgAttendanceRate, totalRefund, refundStudentCount,
    paidLessonRatio, deptSummaries, coachRankings
  }
}
```

## 8. 导出 Excel 设计

导出的 Excel 保留原始 5 个 sheet 结构，并增加：

- **汇总 sheet**（第一个 sheet）：关键指标一页总览
- 其余 sheet 保持原结构，可增加条件格式（如签到率 < 60% 标红）

使用 SheetJS 的 `XLSX.utils.book_new()` + `XLSX.utils.json_to_sheet()` 生成。

## 9. Artifact 输出

作为工具箱中的 Vue 工具，月度分析产出的数据可通过 artifact 系统供其他工具消费：

```typescript
const artifact = {
  name: `月度分析-${fileName}`,
  type: 'monthly-analysis',
  source_tool: 'monthly-analysis',
  data: {
    summary: analysisData.summary,
    schoolAnalysis: analysisData.schoolAnalysis,
    typeAnalysis: analysisData.typeAnalysis,
    coachAttendance: analysisData.coachAttendance,
    refunds: analysisData.refunds,
```

## 11. 与课时费的关系

月度分析和课时费是**独立的两个工具**，在工具箱中并列：

```
工具箱
├── 课时费计算器 (iframe)
│   └── 输入：月度分析表 + 课程类型映射
│   └── 输出：课时费计算结果
│
└── 月度分析 (Vue)
    └── 输入：校内分析 Excel
    └── 输出：仪表盘 + 报告 + Excel + artifact
```

两者可能消费同一个 Excel 文件的不同视角，但各自独立运行，不互相依赖。
