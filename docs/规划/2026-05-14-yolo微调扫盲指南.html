1什么是微调？和重新训练有什么区别？

✅ 微调（Fine-tuning）—— 你该做的

  • 在已有的 YOLOv8n 权重基础上继续训练
  • 只需要 50～500 张 你自己的照片
  • 训练时间 30 分钟 ～ 2 小时（CPU 也能跑）
  • 模型"认识"了你们健身房/教室的场景
  • 精度提升明显，尤其是堆叠和低质照片

❌ 从头训练（Training from Scratch）—— 不必要

  • 需要数十万张标注图片
  • 需要高端 GPU，训练数天
  • 普通业务场景完全没必要
  • 效果不一定比微调好
💡
核心逻辑类比
把 YOLOv8n 想象成一个在全球数百万张照片上训练过的"人类专家"——他认识人、车、动物等80种物体。 你的微调就是给他看你们的具体照片，告诉他："在我们的场景下，拍摄角度是这样的、密度是这样的，你要学会识别这些情况。"
不需要从零教他认识人，只需要让他熟悉你们的场景。

3第一步：数据收集 —— 什么样的照片有用

数量要求

样本数效果预期标注时间（约）推荐程度
50 张 能感受到改善，但不稳定 1～2 小时 最低可用
100～200 张 明显提升，尤其是堆叠和模糊场景 3～6 小时 推荐
500+ 张 效果稳定，接近商业级别 1～3 天 理想

照片质量要求

  • ✅ 覆盖你们的真实场景：不同教室、不同拍摄角度、不同时段光线
  • ✅ 人数分布要均匀：有 0 人（空教室）、少人（3-5 人）、多人（10+ 人）的样本都要有
  • ✅ 问题场景优先：重点收集堆叠/模糊/远景等识别困难的照片
  • ✅ 格式不限：JPG/PNG 均可，不需要统一尺寸
  • ❌ 避免重复帧：同一秒连拍的 10 张几乎没用，不如换 10 个不同场景
  • ❌ 不需要高清原图：手机拍的就够，模型训练用的是 640px 缩放版
⚠️
数据多样性比数量更重要
100 张覆盖各种场景的照片，远好于 500 张同一个角度的照片。 训练数据越接近真实使用时的照片分布，效果越好。

用系统工具收集数据

打开工具箱 → YOLO 模型验证与调参 → 上传照片运行识别 → 在框明细里确认/修正标注 → 点击"保存为训练样本" → 切换到"训练样本库"Tab 导出。

这个流程自动帮你把图片和 YOLO 格式的标注文件存到 data/training_samples/，导出时打包成可以直接训练的 zip。

4第二步：数据标注 —— 手把手教你打标签

什么是 YOLO 格式标注？

每张图片对应一个同名的 .txt 文件，每行描述一个检测框：

labels/photo_001.txt 
# 格式：类别id  中心x  中心y  宽度  高度（所有值都是归一化的 0~1）
0 0.512 0.341 0.120 0.380   # 第1个人
0 0.782 0.510 0.095 0.290   # 第2个人
0 0.251 0.620 0.108 0.350   # 第3个人
ℹ️
归一化是什么意思？
假设图片宽 1920px，一个人的框从像素 x=600 到 x=800，那么：
中心 x = (600+800)/2 / 1920 = 700 / 1920 ≈ 0.365
宽度 = (800-600) / 1920 = 200 / 1920 ≈ 0.104
系统导出时会自动帮你做这个换算，不需要手算。

LabelImg 标注流程（最简单方案）

1
安装 LabelImg
pip install labelImg，然后命令行输入 labelImg 启动。
2
切换到 YOLO 格式
左侧菜单 → 点击 PascalVOC 按钮切换为 YOLO 格式（很重要！）。
3
打开图片目录
File → Open Dir → 选择你的照片文件夹。File → Change Save Dir → 指定标注输出目录。
4
创建标注文件
在 Classes 列表只写 person 一个类别。按 W 键开始画框，框住每一个人（包括被遮挡的），完成后按 D 切换下一张。
⚠️
标注规范 3 条铁律
1. 宁漏勿错：不确定是不是人的，不画框。画了错误框比漏掉一个人危害更大。
2. 堆叠也要画：两个人重叠，每个人单独画一个框，哪怕框之间高度重叠。
3. 至少框住头到腰：全身框最好，但半身框也可以。不要只框头部。

6第四步：训练命令详解

基础训练命令（直接能用）

Terminal 
yolo train \
  model=yolov8n.pt \          # 基础模型：从 yolov8n 微调
  data=dataset/data.yaml \   # 你的数据集配置文件
  epochs=50 \                 # 训练轮数
  imgsz=640 \                 # 训练图片尺寸
  batch=8 \                   # 每次同时训练的图片数
  patience=10 \              # 早停：10 轮无改善就停止
  project=runs/finetune \   # 结果保存目录
  name=person_v1             # 实验名称

参数逐个解析

参数作用调整建议
model 起点模型权重 用 yolov8n.pt（最小最快），效果不好换 yolov8s.pt
epochs 整个数据集被扫描几遍 样本少（<100）用 50；样本多（500+）可到 100。不用手动停，有 patience
imgsz 训练时图片缩放到的边长 默认 640。如果照片里人很小（远景）可试 960，但训练更慢
batch 每次并行处理的图片数 Mac M 系列用 16；CPU 训练用 4；显存 8G GPU 用 16～32
patience 早停：连续 N 轮无改善就停止 10～15。防止过拟合，也节省时间
lr0 初始学习率 默认 0.01。微调时可降到 0.001，让模型更平稳地适应新数据
freeze 冻结前 N 层不参与训练 样本极少（<50）时可用 freeze=10，只训练后几层，防过拟合
augment 数据增强 默认开启，自动翻转、亮度调整，相当于"扩充"数据集

不同场景的推荐配置

🧪 样本极少（50 张以下）

yolo train model=yolov8n.pt data=dataset/data.yaml \
  epochs=30 batch=4 imgsz=640 patience=8 \
  freeze=10 lr0=0.001   # 冻结前10层 + 低学习率，防过拟合

✅ 标准微调（100～300 张）

yolo train model=yolov8n.pt data=dataset/data.yaml \
  epochs=50 batch=8 imgsz=640 patience=10

🚀 数据充足（500+ 张）

yolo train model=yolov8s.pt data=dataset/data.yaml \
  epochs=100 batch=16 imgsz=640 patience=15 \
  lr0=0.005   # 换 yolov8s（更大模型），效果更好
⚠️
训练中途被中断了怎么办？
用 resume=True 恢复：
yolo train resume model=runs/finetune/person_v1/weights/last.pt

Class     Images  Instances      Box(P          R      mAP50   mAP50-95)
  all         20        145      0.912      0.887      0.931      0.612
指标含义好的范围
box_loss 框位置误差（越小越好） 训练结束时应降到初始值的 40%～60%
cls_loss 分类误差（对单类别不重要） 趋势下降即可
P（Precision） 检测到的框里有多少是真正的人（准确率） 越高越好，>0.8 是好结果
R（Recall） 所有真实的人里有多少被检测到了（召回率） 越高越好，>0.8 是好结果
mAP50 主要评估指标：综合精度（IoU 阈值 0.5） >0.85 = 优秀；0.7～0.85 = 良好；<0.7 = 需改善
mAP50-95 更严格的综合精度（多 IoU 阈值平均） >0.5 是好结果，人数计数场景关注 mAP50 更重要

💡
如何判断训练是否过拟合？
打开 results.png，查看 val/box_loss（验证集损失）曲线：
· 一直下降 = 正常，训练还没结束
· 先降后升 = 过拟合了，使用 best.pt（它是升之前的最好结果）
· 一直不降 = 数据量太少或标注质量差，需要补充数据

8第六步：把新模型接入系统

1
备份旧模型
cp backend/yolov8n.pt backend/yolov8n_backup.pt
2
替换模型权重
cp runs/finetune/person_v1/weights/best.pt backend/yolov8n.pt
3
重启后端服务
停止并重启 uvicorn：uvicorn main:app --reload --port 8003
后端用懒加载，重启后第一次请求时会自动加载新模型。
4
用验证工具测试新模型
打开「YOLO 模型验证与调参」工具，上传测试照片，对比新旧模型的识别效果和精度统计。
5
效果不好？回滚
cp backend/yolov8n_backup.pt backend/yolov8n.pt 然后重启后端。
🔴
更换模型前必须重置 NMS 参数
新模型可能有不同的置信度分布，之前在验证工具里调好的参数（conf_threshold、iou_threshold）需要重新验证调整。 不要直接用旧参数上生产，先用几张已知人数的照片验证效果。

9常见问题 FAQ

Q：训练完了，mAP 比原来低了？

通常是因为数据集太小（<50 张）或训练集/验证集分布不均。可以：

  • 补充更多样的照片
  • 确认标注质量（漏标、错标都会拉低 mAP）
  • 加入 freeze=10，只微调后几层

Q：CPU 训练太慢了，有没有免费 GPU？

可以用 Google Colab 免费 GPU：

  1. 打开 colab.research.google.com，新建笔记本
  2. 菜单 → 运行时 → 更改运行时类型 → GPU（T4）
  3. 上传数据集 zip，然后运行训练命令
  4. 下载 best.pt 到本地

Q：堆叠场景微调后还是不准，怎么办？

YOLOv8 本身对极度堆叠的场景存在天然限制（多个高度重叠的框会被 NMS 合并）。进阶方案：

  • 训练时加大 NMS IoU 阈值数据增强：在训练命令加 iou=0.3
  • 换用 YOLOv8-Crowd 或 CrowdDet 等专门为密集人群设计的模型
  • 或者换思路：从照片计人数换成"人群密度估计"（CSRNet / P2PNet），这类模型专门处理堆叠场景

Q：只有几十张照片，值得训练吗？

值得。但期望要合理：

  • 30～50 张能让模型"认识"你们的拍摄角度和场景，通常 mAP 提升 5%～15%
  • 真正大幅提升（20%+）需要 100 张以上，且标注质量好
  • 可以先用系统工具快速积累——每次业务中用到照片核对时，顺手保存几张，一个月后就有了大量训练数据

Q：标注需要框住完整人体吗？半身、背影也要标吗？

规则很简单：凡是你希望系统计为"1个人"的目标，就画一个框。

  • 半身：画半身框 ✅
  • 背影：画框 ✅（模型见过背影就学会认背影）
  • 极远景小点（小于整图面积 0.3%）：不用画，反正过滤掉 ✅
  • 人形雕像/海报上的人：不画 ✅
