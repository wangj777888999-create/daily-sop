# 工具箱三大工具建设规划

> 日期：2026-05-12
> 状态：待审批

## 一、背景与目标

工具箱（Toolbox）当前仅有月度分析的占位卡片。本规划将三个核心数据分析工具纳入工具箱：

1. **每日教练签到分析** — 每日签到 + 财务数据处理，结果持久化到 SQLite
2. **校内月度分析** — 基于每日签到累积数据 + 财务/退款/课程类型的月度报表
3. **校外月度分析** — 独立的校外校区经营分析（场地费/课时费/到课率等 23 个指标）

三个工具的数据来源为同一 SaaS 平台导出的 Excel 文件，用户手动下载后上传到工具箱处理。

### 工具间联动关系

```
每日签到分析 ──写入──→ SQLite ──读取──→ 校内月度分析
                                        （自动加载签到数据）

校外月度分析 ←──独立── （不依赖前两个工具，自有数据源）
```

## 二、技术方案

### 2.1 方案选择：原生 FastAPI 集成

将现有 Python 分析脚本移植为后端 Python 模块，创建专用 API 端点，SQLite 做持久层。

选择理由：
- 与现有架构一致（SOP 引擎、RAG 知识库都是原生模块）
- 三个工具是长期固定的系统功能，非用户自定义脚本
- SQLite 集成需要在模块级别做，subprocess 方案不适合
- 后续前端可视化（ECharts）需要结构化 JSON 数据

### 2.2 建设顺序

由于校内月度分析依赖每日签到数据，按以下顺序推进：

1. 每日教练签到分析（基础工具，最先建设）
2. 校内月度分析（依赖日签数据的累积）
3. 校外月度分析（独立，可最后或并行）

## 四、SQLite 数据库设计

```sql
-- 每日签到记录（由每日工具写入，月度工具读取）
CREATE TABLE daily_checkin (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    check_date DATE NOT NULL,           -- 签到日期（业务日期，如 2026-04-21）
    department TEXT NOT NULL,           -- 部门
    school_name TEXT NOT NULL,          -- 学校名称
    course_type TEXT,                   -- 课程类型
    course_name TEXT NOT NULL,          -- 课程名称
    coach_name TEXT NOT NULL,           -- 教练姓名
    course_date TEXT,                   -- 课程日期
    start_time TEXT,                    -- 开课时间
    end_time TEXT,                      -- 下课时间
    sign_in_time TEXT,                  -- 签到时间
    sign_out_time TEXT,                 -- 签退时间
    sign_status TEXT NOT NULL,          -- 签到状态（在岗/迟到/早退/旷工）
    actual_count INTEGER DEFAULT 0,     -- 实际上课人次
    expected_count INTEGER DEFAULT 0,   -- 课程应到人次
    confirmed_revenue REAL DEFAULT 0,   -- 确认收入
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(check_date, course_name, coach_name, course_date, start_time)
);

-- 签到数据批次（追踪每次上传来源）
CREATE TABLE checkin_batches (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    batch_date DATE NOT NULL,
    coach_file TEXT,                    -- 原始教练签到文件名
    finance_file TEXT,                  -- 原始财务文件名
    record_count INTEGER DEFAULT 0,     -- 处理记录数
    status TEXT DEFAULT 'completed',    -- completed / failed
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引：月度查询按日期+部门
CREATE INDEX idx_checkin_date_dept ON daily_checkin(check_date, department);
CREATE INDEX idx_checkin_coach ON daily_checkin(coach_name, check_date);
```

## 五、工具一：每日教练签到分析

### 5.1 用户操作流程

```
上传 Excel → 数据预览 → 确认处理 → 存入 SQLite → 下载结果 Excel
```

1. 上传两个 Excel 文件：教练签到 + 财务统计
2. 系统解析并展示前 50 行预览，验证列名和数据格式
3. 用户确认无误后点击"处理"
4. 后端执行分析逻辑，结果写入 SQLite + 生成格式化 Excel
5. 用户可下载带高亮的 Excel，数据同时持久化供月度使用

### 5.2 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/tools/daily-checkin/preview` | 上传 Excel，返回前 50 行预览 + 列名验证结果 |
| POST | `/api/tools/daily-checkin/process` | 执行分析，写入 SQLite，返回结果摘要（记录数/异常数） |
| GET | `/api/tools/daily-checkin/download/{batch_id}` | 下载格式化 Excel（含黄色高亮） |
| GET | `/api/tools/daily-checkin/history` | 返回历史处理记录列表 |

### 5.3 后端处理逻辑

移植自 `samples/教练签到分析最终版.py`，核心函数：

| 原函数 | 新模块函数 | 说明 |
|--------|-----------|------|
| `process_coach_data()` | `parse_coach_data(df)` | 筛选"学校课程"，跳过前两行，清理列名 |
| `process_finance_data()` | `parse_finance_data(df)` | 跳过前三行，按课程聚合"实际上课人次/应到人次/确认收入" |
| `merge_and_reorder_data()` | `merge_all(coach, dept, finance)` | 三表合并，调整列顺序（部门→其他→教练→三指标） |
| `export_with_formatting()` | `generate_excel(df) -> bytes` | 生成带居中对齐+黄色高亮+自适应列宽的 Excel |

新增逻辑：
- `save_to_sqlite(df, batch_info)` — 将处理结果写入 `daily_checkin` 表
- 去重写入（利用 `UNIQUE` 约束 + `INSERT OR IGNORE`）
- 部门划分表从 `data/config.json` 配置路径读取，不再每次上传

### 5.4 前端页面组件

- 文件上传区：拖拽或点击上传两个 Excel（教练签到 + 财务统计）
- 预览表格：使用 `DataTable` 展示解析后的前 50 行，支持滚动
- 验证状态：列名校验通过/失败的标识
- 结果表格：签到状态不为"在岗"的行黄色高亮
- 操作按钮：上传 / 预览 / 确认处理 / 下载 Excel
- 历史记录：`DataTable` 展示 `checkin_batches` 记录

## 六、工具二：校内月度分析

### 6.1 用户操作流程

```
选择月份 → 自动加载签到数据 → 上传补充文件 → 生成报告 → 预览 5 个 Sheet → 下载
```

1. 月份选择器选择目标月份
2. 系统自动从 SQLite 读取该月所有签到数据
3. 用户上传补充 Excel：财务明细、课程类型映射、退款数据
4. 点击"生成报告"，后端执行 4 项分析
5. 前端以 Tab 页展示 5 个 Sheet 的数据预览
6. 下载合并的 Excel 文件

### 6.2 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/tools/campus-monthly/available-months` | 返回 SQLite 中有签到数据的月份列表 |
| POST | `/api/tools/campus-monthly/preview` | 上传补充文件，返回签到数据预览 + 文件验证 |
| POST | `/api/tools/campus-monthly/analyze` | 执行分析，返回 5 个 Sheet 的 JSON 数据 |
| GET | `/api/tools/campus-monthly/download` | 下载结果 Excel（5 个 Sheet） |

### 6.3 后端处理逻辑

移植自 `samples/campus_monthly_analysis.py`，4 项分析：

| 分析项 | 函数 | 输入数据 | 输出 Sheet |
|--------|------|---------|-----------|
| 校内财务分析 | `campus_finance_analysis()` | 签到数据(从 SQLite) + 财务明细(上传) | 校内分析：按部门/学校/教练/课程汇总上课人次、次数、确认收入 |
| 教练考勤分析 | `coach_checkin_analysis()` | 签到数据(从 SQLite) | 教练考勤：个人签到率、学校签到率、部门签到率 |
| 课程类型分析 | `course_type_analysis()` | 签到数据(从 SQLite) + 课程类型映射(上传) | 类型分析：体育课/兴趣班/校队/梯队/社团课分布 + 收费类占比 |
| 退款分析 | `refund_analysis()` | 退款数据(上传) + 部门划分(系统配置) | 退款分析：按部门/学校汇总退款人数和金额 |
| 原始签到 | 直接透传 | 签到数据(从 SQLite) | 教练签到：全量签到记录 |

关键差异（对比原脚本）：
- 签到数据从 SQLite 读取而非上传 Excel
- 部门划分、教练名单等基础数据从系统配置读取
- 财务明细和退款数据仍需用户上传（非每日流程产出）

### 6.4 前端页面组件

- 月份选择器：下拉选择月份，自动显示可用月份
- 签到数据概览：显示该月签到记录数、涉及教练数、学校数
- 文件上传区：财务明细 Excel + 课程类型 Excel + 退款 Excel
- 5 个 Tab 页：每个 Tab 对应一个 Sheet，使用 `DataTable` 展示数据
- 下载按钮：导出完整 5-Sheet Excel

## 七、工具三：校外月度分析

### 7.1 用户操作流程

```
上传文件 → 校内外拆分预览 → 确认分析 → 综合统计 → 下载
```

1. 上传 4 个 Excel：上课记录、校外课时费、财务数据、场地使用费用汇总
2. 系统自动将上课记录拆分为校外/校内，预览拆分结果
3. 确认后执行完整分析（12 个子步骤）
4. 展示综合统计结果（按校区-教练维度，23 个指标）
5. 下载两个 Excel：主文件（筛选后列）+ 完整数据

### 7.2 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/tools/offcampus-monthly/preview` | 上传文件，返回校内外拆分统计预览 |
| POST | `/api/tools/offcampus-monthly/analyze` | 执行完整分析，返回综合统计 + 完整数据 |
| GET | `/api/tools/offcampus-monthly/last-month` | 获取上月分析结果（用于环比计算） |
| GET | `/api/tools/offcampus-monthly/download` | 下载 Excel（主文件 + 完整数据两个 Sheet） |

### 7.3 后端处理逻辑

移植自 `samples/offcampus_monthly_analysis.py`，12 个子步骤：

| 步骤 | 函数 | 说明 |
|------|------|------|
| 1 | `skjl_separate()` | 按课程名关键词（农都/西溪/滨江）拆分校内外上课记录 |
| 2 | `process_finance_data()` | 处理财务数据，筛选"球馆学员"，生成课程-教练映射 |
| 3 | `calculate_basic_stats()` | 按"校区-教练"统计上课节次、人次、平均课堂人次 |
| 4 | `calculate_attendance_rate()` | 计算到课率（排除私教课程） |
| 5 | `calculate_revenue()` | 统计确认收入、平均课程单价 |
| 6 | `analyze_class_size()` | 5 人及以下 / 10 人及以上课程数及比例 |
| 7 | `calculate_campus_contribution()` | 教练对本校区的营收贡献比 |
| 8 | `calculate_offcampus_in_campus()` | 统计校外教练在校内的上课节次 |
| 9 | `merge_teaching_fee()` | 合并校外课时费数据 |
| 10 | `site_cost()` | 场地费计算（含校区生均、环比、成本占比） |
| 11 | `summarize_campus()` | 添加校区汇总行 |
| 12 | `summarize_total()` | 添加全部汇总行 |

输出 23 个指标列：

| 维度 | 指标 |
|------|------|
| 基础 | 校区、教练、课次、上课人次、平均课堂人次、到课率 |
| 收入 | 确认收入、平均课程单价、本校区营收贡献比 |
| 成本 | 场地费、应发课时费、S3:场地费生均成本、(场地费+课时费)/确收 |
| 班级规模 | 5 人及以下课程数/比例、10 人及以上课程数/比例 |
| 环比 | 上月场地费生均成本、S3 较上月变化 |
| 场地效率 | 低于校区生均场地费占比、高于校区生均场地费 1 倍占比、最高课程生均场地费/S3 |
| 跨校区 | 校内上课节次 |

### 7.4 前端页面组件

- 文件上传区：4 个 Excel 文件上传（上课记录、课时费、财务、场地费用）
- 拆分预览：显示校外/校内记录数、涉及校区数
- 上月对比：自动加载上月分析结果，显示环比数据
- 综合统计表：使用 `DataTable` 展示 23 个指标，按校区分组
- 校区汇总/总计行：灰色背景标识汇总行
- 下载按钮：导出主文件 + 完整数据两个 Excel
